# CoralStack Infrastructure

Reproducible, self-hosted service stack. Sign-in-with-Apple for people who'd rather own it.

This repo is the distributable artifact behind [coralstack.org](https://coralstack.org) — a local
cooperative model for digital infrastructure. One co-op member hosts on commodity hardware; the rest
of the community connects over Tailscale or public DNS. Any community can fork and deploy.

## What it runs

| Service      | Purpose                   | Replaces                |
| ------------ | ------------------------- | ----------------------- |
| Pocket ID    | OIDC + passkeys (SSO)     | Sign in with Apple      |
| Immich       | Photos & videos           | iCloud Photos           |
| Jellyfin     | Music (+ movies/TV)       | Apple Music / Spotify   |
| Vaultwarden  | Password vault            | iCloud Keychain         |
| Caddy        | Reverse proxy + auto-TLS  | Manual cert juggling    |

Services are opt-in: comment any service out of `docker-compose.yml` (the top-level `include:` list)
and it won't run. Pocket ID is the one service worth keeping on if you want SSO across the rest.

## Quickstart

```bash
git clone <this-repo> coralstack-infra
cd coralstack-infra
cp .env.example .env
# edit .env — set COMMUNITY, BASE_DOMAIN, STORAGE_PATH
./setup.sh
```

`setup.sh` checks prereqs, generates per-service secrets, and brings the stack up.

See [docs/QUICKSTART.md](docs/QUICKSTART.md) for the full first-run walkthrough and
[docs/ONBOARDING.md](docs/ONBOARDING.md) for the OIDC wiring steps after first boot.

## Architecture

```
                         ┌──────────────┐
    tailnet / public ──▶│    Caddy     │── auto-TLS, routes by subdomain
                         └──────┬───────┘
                                │
         ┌──────────┬───────────┼───────────┬──────────┐
         ▼          ▼           ▼           ▼          ▼
     Pocket ID   Immich     Jellyfin   Vaultwarden  (future)
        (OIDC provider)
```

Subdomain convention: `{service}.{BASE_DOMAIN}`. E.g. `photos.campbellriver.coralstack.org`.

## Design notes

- **Compose-per-service.** Each service owns `services/<name>/docker-compose.yml`. The top-level
  `docker-compose.yml` pulls them in via `include:` so you can pick your combo.
- **Single shared `.env`** at the repo root for community-wide config (domain, storage path, TZ).
  Per-service secrets live in `services/<name>/.env` and are generated by `setup.sh`.
- **Caddy is the TLS boundary.** Real Let's Encrypt certs via Cloudflare DNS-01 — the NUC never
  needs to be publicly reachable. DNS for your domain has to be hosted at Cloudflare; see
  [QUICKSTART.md](docs/QUICKSTART.md#dns-setup) for the ~10-minute nameserver swap if you're
  elsewhere. Wildcard-friendly: one cert covers every subdomain.
- **Pocket ID is the identity root.** One passkey → SSO into every other service. The OIDC client
  configuration is a manual step after first boot — `setup.sh` generates the secrets and prints
  the exact admin-panel clicks.
